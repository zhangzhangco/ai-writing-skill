# 素材库管理示例

本示例展示如何高效管理个人素材库，这是降AI味的核心策略。

## 素材库四大类型

### 1. 经历类素材

**定义**：个人真实经历、观察、体验

**使用场景**：
- 增强文章可信度
- 提供第一手资料
- 建立读者信任

**管理示例**：

```typescript
// 添加个人经历
await tools['manage-corpus'].handler({
  action: 'add',
  material_data: {
    title: '2023年HDR拍摄实战经历',
    content: `
2023年国庆期间，我在青海湖用iPhone 15 Pro拍摄日出。

实际体验：
- HDR自动模式下，云层细节保留完整
- 日出色彩过渡自然，无过曝现象
- 功耗确实比普通模式高约20%
- 连续拍摄2小时后设备温热

关键技术细节：
- A17 Pro的ISP处理延迟约150ms
- 动态范围提升约3EV
- 噪点控制优于上一代约30%
    `,
    tags: ['HDR', 'iPhone', '青海湖', '日出', '实战'],
    category: '经历'
  }
});

// 搜索经历类素材
const experiences = await tools['manage-corpus'].handler({
  action: 'search',
  keywords: 'HDR实战'
});
```

**素材库统计**：
```
经历类素材：25个
- 按标签分布：
  - HDR：8个
  - 摄影：12个
  - 设备评测：5个
```

### 2. 观点类素材

**定义**：个人独特观点、见解、分析框架

**使用场景**：
- 提供差异化视角
- 增强文章深度
- 建立专业权威

**管理示例**：

```typescript
await tools['manage-corpus'].handler({
  action: 'add',
  material_data: {
    title: 'HDR技术发展三阶段论',
    content: `
我的观点：HDR技术将经历三个发展阶段

第一阶段：硬件主导（2020-2023）
- 特点：硬件HDR处理器主导
- 优势：处理速度快
- 局限：能耗高、成本贵

第二阶段：软硬协同（2023-2026）
- 特点：硬件加速 + 软件算法优化
- 优势：平衡性能与成本
- 典型：iPhone的计算摄影

第三阶段：AI原生（2026+）
- 特点：AI算法主导，硬件为辅
- 优势：智能优化、自适应
- 趋势：端云协同HDR

这个三阶段论可以帮助我们判断技术成熟度和投资时机。
    `,
    tags: ['HDR', '技术分析', '趋势预测', '三阶段论'],
    category: '观点'
  }
});
```

### 4. 风格类素材

**定义**：个人写作风格、表达习惯、语言特色

**使用场景**：
- 保持风格一致
- 降AI腔
- 建立个人品牌

**管理示例**：

```typescript
await tools['manage-corpus'].handler({
  action: 'add',
  material_data: {
    title: '科技写作风格指南',
    content: `
我的写作风格特征：

1. 数据先行
- 永远用数据说话
- 提供具体数字而非形容词
- 例：不是"很多"，而是"15个"

2. 体验为王
- 偏好第一手体验
- 真实感受 > 官方说法
- 例：实际拍摄发现功耗确实比官方声称高5%

3. 逻辑递进
- 从问题到方案到效果
- 3段式结构（问题-分析-解决）
- 每段3-5个要点

4. 语言特色
- 口语化表达，避免教科书式语言
- 适当使用反问句引发思考
- 善用对比突出差异

5. 反AI腔清单
- 避免："值得注意的是"
- 避免："综上所述"
- 避免："不难发现"
- 用具体事实替代模糊表达
    `,
    tags: ['风格', '写作指南', '反AI腔', '科技写作'],
    category: '风格'
  }
});
```

## 素材库使用策略

### 1. 写作前：素材盘点

```typescript
// 写作前检查相关素材
const relevantMaterials = await tools['manage-corpus'].handler({
  action: 'search',
  keywords: 'HDR iPhone 移动设备'
});

console.log('可用素材:', relevantMaterials.results.materials.length);
console.log('素材类型分布:', relevantMaterials.statistics.by_type);
```

**输出示例**：
```
可用素材: 18个
素材类型分布:
- 经历: 5个
- 观点: 3个
- 案例: 7个
- 风格: 3个
```

### 2. 写作中：实时调用

**调用原则**：
- 每500字至少1个素材引用
- 优先使用经历类素材（最真实）
- 关键数据必须有案例支撑
- 保持风格一致性

**调用示例**：
```markdown
HDR技术在移动设备上确实面临功耗挑战。（观点引用）

从我实际测试iPhone 15 Pro的经历来看，连续HDR拍摄30分钟，电量消耗约15%，比普通模式高5个百分点。（经历引用）

功耗测试数据显示：
- iPhone 15 Pro：30分钟耗电15%
- Galaxy S24 Ultra：30分钟耗电18%
（案例引用）

这符合"硬件主导阶段"的技术特征，硬件ISP的能耗是制约因素。（观点引用）
```

### 3. 写作后：素材优化

```typescript

const corpus = await tools['manage-corpus'].handler({

console.log('素材库使用率:');
```

**目标**：
- 素材调用率 ≥ 80%
- 每类素材都有使用
- 新素材生成 ≥ 3个

## 素材库扩展策略

### 1. 日常积累

- **随时记录**：好记性不如烂笔头
- **分类管理**：按标签系统归类
- **定期更新**：每周新增3-5个素材

### 2. 专题收集

针对特定主题，系统性收集：
- 技术标准文档
- 实际测试数据
- 行业报告
- 用户反馈

### 3. 质量提升

- 定期审核：删除过时、错误素材
- 丰富内容：增加细节、标签
- 建立关联：素材间的引用关系

## 素材库与降AI味

**核心原理**：
AI生成的内容偏向通用、抽象、无个人特色
个人素材库提供：
- 真实经历
- 独特观点
- 具体数据
- 个人风格

**降AI腔三原则**：
1. **数据 > 观点**：用具体数字替代模糊表达
2. **经历 > 理论**：用个人体验支撑论点
3. **个性 > 标准**：用独特语言替代模板

**效果验证**：
- AI腔比例 < 2%
- 素材调用率 > 80%
- 读者反馈："像真人写的"

## 最佳实践

1. **从第一篇开始建库**：不要等到有积累再开始
2. **宁滥勿缺**：初期多收集，后期可精炼
3. **持续更新**：素材库是动态资产，需要维护
4. **定期回顾**：每月检查素材使用情况
5. **开放心态**：接受新观点，更新旧认知

通过高效的素材库管理，您的文章将具备独特的个人色彩，有效降低AI腔，提升内容质量。
