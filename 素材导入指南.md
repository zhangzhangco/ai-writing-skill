# 个人素材导入指南

本文档详细介绍如何将您的个人素材导入到 ai-writing-skill 中。

---

## 🎯 素材库四大类型

### 1️⃣ **经历类** - 真实项目经验
- **内容**：您亲自参与的项目、实战经验、踩过的坑
- **使用场景**：文章开篇引入、论证支撑、反面教材
- **示例**：
  - "2023年我负责的HDR显示技术研究项目"
  - "在开发数字水印平台时遇到的性能问题"
  - "第一次使用AI写作工具的尝试经历"

### 2️⃣ **观点类** - 独立思考判断
- **内容**：您对行业、技术、趋势的独特观点
- **使用场景**：抛出观点、论证支撑、转折过渡
- **示例**：
  - "我认为HDR技术会经历三个发展阶段"
  - "写作是思考的外化，不是文字堆砌"
  - "AI是工具，不是替身"

### 3️⃣ **案例类** - 实测数据案例
- **内容**：具体的测试数据、对比实验、验证案例
- **使用场景**：数据支撑、案例说明、对比分析
- **示例**：
  - "300nit vs 600nit亮度对比实测数据"
  - "10款AI写作工具功能对比表"
  - "某品牌激光投影仪与氙灯投影仪的测试"

### 4️⃣ **风格类** - 写作风格特征
- **内容**：您的语言习惯、写作偏好、禁用表达
- **使用场景**：风格对齐、审校参考、保持一致性
- **示例**：
  - "偏好用'我'而非'我们'开头"
  - "避免使用'值得注意的是'等套话"
  - "喜欢用具体数据而非形容词"

## 📝 如何导入素材

### 方式1：直接告诉Claude（推荐）

在Claude中直接说：
```
"请帮我添加一个素材到素材库"

参数：
- 素材类型：经历 / 观点 / 案例 / 风格
- 标题：[一句话概括]
- 内容：[详细描述]
- 标签：[相关标签1, 标签2, 标签3]
- 来源：[项目名称/时间/文件]
```

**具体示例**：

**添加经历类素材**：
```
"添加一个经历类素材"
- 标题：HDR显示技术研究项目实战
- 内容：2023年我在XX公司负责HDR显示技术研究。项目历时6个月，我们测试了5种不同的激光方案，发现......（详细经历）
- 标签：HDR, 激光显示, 实战经验
- 来源：XX公司项目，2023.06-2023.12
```

**添加观点类素材**：
```
"添加一个观点类素材"
- 标题：HDR技术三阶段论
- 内容：我认为HDR技术会经历三个发展阶段：第一阶段是技术验证期（2015-2020），第二阶段是成本优化期（2021-2024），第三阶段是普及应用期（2025-2030）。这一判断基于......
- 标签：HDR, 技术发展, 趋势预测
- 来源：个人思考，2024.11
```

**添加案例类素材**：
```
"添加一个案例类素材"
- 标题：激光vs氙灯投影实测对比
- 内容：我们实测了某品牌激光投影仪与氙灯投影仪的对比数据。测试条件：......测试结果：亮度对比6500流明 vs 6000流明，色彩还原120% NTSC vs 80% NTSC，寿命25000小时 vs 800-1500小时......
- 标签：激光投影, 氙灯, 测试数据
- 来源：项目测试报告，2024.10
```

**添加风格类素材**：
```
"添加一个风格类素材"
- 标题：我的写作风格特征
- 内容：1. 偏好用'我'而非'我们'开头 2. 避免'值得注意的是'、'综上所述'等套话 3. 喜欢用具体数字而非'很多'、'很多'等模糊表达 4. 句式长度控制在20-30字 5. 每段1-2个信息点
- 标签：写作风格, 个人习惯, 语言特征
- 来源：个人总结，2024.11
```

## 🛠️ 方式2：批量导入

如果素材较多，可以批量添加：
```
"我有很多素材想批量导入素材库，包括：
1. 2023年HDR项目经历
2. 关于AI写作的观点
3. 激光投影测试数据
4. 我的写作风格

请逐个帮我添加，顺序是经历→观点→案例→风格"
```

## 📋 素材格式要求

### 推荐格式
```markdown
**标题**：[简洁描述]

**内容**：
[详细描述，200-500字]

**关键信息**：
- 时间：[YYYY.MM]
- 地点：[项目/公司/场景]
- 数据：[具体数字]
- 结果：[得出的结论]

**标签**：[标签1, 标签2, 标签3]

**备注**：[额外说明]
```

### 质量检查清单
- [ ] 内容真实可信
- [ ] 有具体细节（时间、地点、数据）
- [ ] 观点独特有见地
- [ ] 案例有验证数据
- [ ] 风格描述清晰
- [ ] 标签准确易搜索

## 🎨 导入示例

### 完整示例1：技术经历
```
标题：数字水印平台开发项目
内容：
2023年Q2，我负责公司数字水印平台的架构设计。项目背景是应对AI生成内容的版权保护问题。

技术挑战：
1. 实时性要求高（<100ms响应）
2. 嵌入强度与视觉质量的平衡
3. 多格式兼容（JPEG/PNG/视频）

解决方案：
- 采用DCT域嵌入算法
- 自适应强度调整
- 分布式处理架构

最终成果：
- 处理速度提升300%
- 视觉质量评分4.5/5
- 支持12种格式

经验总结：
- 实时性优化要从底层算法入手
- 用户体验与技术实现要平衡
- 分布式架构要考虑一致性

关键数据：处理速度3000张/秒，原系统仅750张/秒

标签：数字水印, 实时处理, 架构设计
来源：XX公司项目，2023.04-2023.08
```

### 完整示例2：观点
```
标题：AI写作的三个层次
内容：
我认为AI写作可以分为三个层次：

第一层：工具层
把AI当作高级搜索和整理工具，辅助查找资料、组织信息。这是目前大多数人的用法。

第二层：协作层
与AI形成真正的协作关系，AI负责信息处理和文字润色，人负责观点和逻辑。这是理想状态。

第三层：思维层
AI成为思考的延伸，帮助发现思维的盲点，拓展认知边界。这是未来方向。

判断标准：
- 是否保持了独立思考？
- 是否产出了独特观点？
- 是否提升了思维质量？

目前我处于第二层向第三层过渡阶段。

标签：AI写作, 协作模式, 思维层次
来源：个人思考，2024.11
```

## 💡 最佳实践

### 1️⃣ **定期更新**
- 每次项目结束后添加经历
- 每月添加新观点
- 每季度更新风格库
- 测试后添加案例

### 2️⃣ **质量优先**
- 宁缺毋滥，只添加高质量素材
- 有具体数据优先
- 独特观点优先
- 真实经历优先

### 3️⃣ **标签规范**
- 使用统一的标签体系
- 标签数量3-5个为宜
- 包含主题、技术、行业等维度

### 4️⃣ **使用循环**
添加 → 搜索 → 使用 → 反馈 → 优化

---

## ⚠️ 注意事项

1. **真实性第一** - 所有素材必须真实，不要编造
2. **细节丰富** - 越具体越好（时间、地点、数据、感受）
3. **观点独特** - 避免千篇一律，要有个人见解
4. **格式规范** - 保持统一的格式便于搜索
5. **定期维护** - 删除过时、错误的素材
