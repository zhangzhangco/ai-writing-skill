# 中文写作标准更新

## 📋 更新内容

根据用户要求，已更新skill的中文写作标准，从**初稿阶段**就确保质量。

## ✅ 新增检查标准

### 1. 段落长度标准
- **要求**：每段200-500字
- **原因**：符合中文阅读习惯，长段落更自然
- **检查**：fluency-optimizer会检测短段落(<200字)和超长段落(>500字)
- **在初稿中的应用**：初稿生成时强制使用200-500字段落

### 2. 问句控制标准
- **要求**：问句比例<1%，技术文章避免问句
- **原因**：问句会影响技术文章的专业性和严肃性
- **检查**：检测疑问词开头、以"？"结尾的句子
- **在初稿中的应用**：初稿生成时避免使用疑问句

### 3. 段落排版标准
- **要求**：连贯段落排版，避免诗式换行
- **原因**：技术文章应该连贯流畅，而非碎片化
- **检查**：检测连续的短行（<50字）合并问题
- **在初稿中的应用**：相关内容必须合并为完整段落

### 4. 句长变化标准
- **要求**：10字短句 + 30字中句 + 50字长句混合
- **原因**：节奏变化提升可读性
- **检查**：分析句子长度分布
- **在初稿中的应用**：自动控制句长比例

## 🛠️ 工具更新

### fluency-optimizer.ts
新增3个分析维度：

1. **chinese_paragraph_length** - 中文段落长度检查
   - 短段落：<200字 → 建议扩展
   - 超长段落：>500字 → 建议拆分

2. **question_check** - 问句检查
   - 检测疑问词开头句子
   - 检测以"？"结尾的句子
   - 问句比例>1%扣分

3. **poetic_line_break** - 诗式换行检查
   - 检测连续的短行（<50字）
   - 建议合并为连贯段落

### run-workflow.ts
在初稿生成阶段添加约束：
- 段落必须200-500字
- 避免使用问句
- 相关内容合并为段落
- 句长适度变化

### review-article.ts
第四遍审校增强：
- 检查所有新增标准
- 提供具体修改建议
- 必要时要求重写

## 📊 质量门槛

| 检查项 | 合格标准 | 不合格后果 |
|--------|----------|------------|
| 段落长度 | 200-500字 | 重写段落 |
| 问句比例 | <1% | 修改为陈述句 |
| 诗式换行 | <5处 | 合并为段落 |
| 长句比例 | <5% | 拆分长句 |
| 流畅度评分 | ≥4.0 | 继续优化 |

## 🎯 实施效果

更新后的skill能够：
1. **初稿阶段**就生成符合中文习惯的连贯文章
2. **质量控制**时全面检查并修正问题
3. **确保**每篇文章都具备专业性和可读性

## 📝 使用建议

1. **初稿阶段**：严格按照200-500字段落生成
2. **审校阶段**：使用fluency-optimizer进行全面检查
3. **优化阶段**：根据检查结果逐项修正
4. **验证阶段**：确保所有标准达标才可发布
